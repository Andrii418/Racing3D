Detailed Description

Główny plik gry: inicjalizacja systemów, pętla gry, renderowanie i UI.

Plik łączy w sobie:

  • inicjalizację GLFW/GLAD (OpenGL 3.3 Core),
  • inicjalizację ImGui (interfejs),
  • inicjalizację audio (miniaudio),
  • logikę stanów gry (splash -> menu -> wyścig),
  • aktualizację fizyki i AI,
  • renderowanie sceny do FBO (postprocess blur w menu),
  • renderowanie HUD/menus.
Note
Jest to „plik-spoiwo”. W przyszłości warto wydzielić moduły: audio, UI, gameplay, rendering.

◆ main()

int main ( )

Entry point aplikacji.

Fazy startu: 1) init GLFW + create window, 2) load GLAD, 3) init ImGui, 4) init render zasobów (shaders/FBO/skybox), 5) load profilu + init audio, 6) init obiektów świata (car/ai/camera/tracks), 7) pętla gry.

Dodatkowe informacje:

  • OpenGL działa w trybie Core Profile 3.3, więc wymagane są VAO/VBO oraz własne shadery.
  • Renderowanie odbywa się dwuetapowo: scena -> FBO -> quad fullscreen (postprocess).
  • ProfileManager jest ładowany na starcie, by odtworzyć saldo i odblokowane auta.
Returns
Kod zakończenia procesu (0 oznacza poprawne zakończenie).

Inicjalizacja GLFW.

GLFW zapewnia:

  • tworzenie okna,
  • obsługę kontekstu OpenGL,
  • event loop (input, resize, itp.).

Jeśli inicjalizacja nie powiedzie się, nie ma sensu kontynuować.

Ustawienia kontekstu OpenGL.

  • 3.3: wersja kontekstu,
  • Core profile: brak funkcji „fixed pipeline” (compatibility), wszystko na shaderach.

Tworzenie okna i kontekstu.

Parametry:

  • current_width/current_height: początkowy rozmiar okna,
  • "Racing3D": tytuł okna,
  • brak fullscreen (nullptr),
  • brak współdzielenia zasobów (share window = nullptr).

Aktywacja kontekstu OpenGL dla bieżącego wątku.

W OpenGL kontekst jest powiązany z wątkiem — wywołania GL muszą być wykonywane w wątku, gdzie kontekst jest „current”.

Rejestracja callbacków GLFW.

  • framebuffer_size_callback: aktualizacja viewportu i rekonstrukcja FBO po resize,
  • key_callback: obsługa klawiatury (m.in. ESC i przełączenie kamery).

Inicjalizacja GLAD (ładowanie wskaźników funkcji OpenGL).

Bez GLAD wywołania funkcji OpenGL (np. glGenBuffers) nie będą dostępne.

Włączenie testu głębokości.

W trakcie renderowania sceny 3D (do FBO) test głębokości jest wymagany, aby obiekty zasłaniały się poprawnie.

Inicjalizacja ImGui.

  • ImGui::CreateContext() tworzy globalny kontekst UI,
  • backend GLFW obsługuje input i okna,
  • backend OpenGL3 wykonuje właściwe rysowanie.

Styl ImGui.

Ustawienia wyglądu UI: zaokrąglenia, kolory, obramowania. Dzięki temu UI ma spójny wygląd w całej aplikacji.

Inicjalizacja shaderów sceny i post-processingu.

  • carTrackShader obsługuje renderowanie aut i świata (Phong),
  • postProcessShader obsługuje render quada fullscreen oraz blur w menu/splash.
Note
Shader postProcessShader oczekuje screenTexture = 0 (GL_TEXTURE0).

Zasoby post-processingu.

  • setupPostProcessingQuad() tworzy VAO/VBO quada fullscreen,
  • setupFramebuffer() buduje FBO o rozmiarze zgodnym z oknem.

Konfiguracja stb_image.

Ustawienie false oznacza, że obrazki nie są odwracane w pionie przy ładowaniu. Zależy to od tego, jak są przygotowane UV w modelach / shaderach.

Inicjalizacja geometrii skyboxa (kostka).

Skybox renderujemy jako prostą kostkę bez przesuwania względem kamery (tylko rotacja view).

Ścieżki do tekstur cubemapy skyboxa.

Kolejność zgodna z OpenGL: +X, -X, +Y, -Y, +Z, -Z.

Załadowanie cubemapy i utworzenie programu shaderów skyboxa.

Wczytanie tekstury tła (splash/menu).

Wczytanie profilu gracza (saldo, odblokowane auta, wybrane auto).

Jeśli save.txt nie istnieje, ProfileManager powinien utworzyć profil domyślny.

Inicjalizacja audio (miniaudio).

  • inicjalizujemy audio_engine,
  • ładujemy pętlę dźwięku silnika,
  • startujemy dźwięk od razu, ale na głośności 0 (będzie sterowana w runtime).

Tworzenie obiektu samochodu gracza.

Pozycja startowa jest tymczasowa — faktyczne ustawienie pozycji wyścigu następuje po wciśnięciu START RACE w menu.

Ładowanie assetów samochodu na podstawie profilu.

Szukamy w garage auta o id playerProfile.currentCarId. Jeśli nie znajdziemy lub ładowanie się nie powiedzie, fallbackujemy na pierwszy wpis.

Tworzenie samochodu AI.

AI na starcie jest ustawione w pobliżu auta gracza, a następnie jego pozycja jest resetowana przy starcie wyścigu.

Note
AI zawsze używa obecnie modelu "race.obj".

Waypointy AI.

Lista punktów definiuje pętlę jazdy AI (tor). AI przechodzi przez punkty w kolejności, po osiągnięciu końca zawija do 0.

Note
To jest „twardo wpisana” trasa. Docelowo można to przenieść do pliku/edytora.

Ograniczenie prędkości AI względem gracza.

Dzięki temu AI jest minimalnie wolniejsze i wyścig jest „do wygrania”.

Inicjalizacja kamery i sceny.

  • Kamera startuje w trybie chase (pozycja początkowa to tylko default),
  • track to arena,
  • city i kartingMap zależą od wyboru trasy.

Inicjalizacja systemu kolizji toru.

Parametr (2.0f) jest skalą/konfiguracją używaną przez TrackCollision.

Główna pętla aplikacji.

Kluczowe kroki każdej klatki:

  • obliczenie deltaTime,
  • aktualizacja logiki gry zależnie od currentState,
  • render sceny do FBO,
  • render postprocess na ekran,
  • render UI (ImGui),
  • swap buffers + poll events.

Obliczenie czasu klatki.

deltaTime jest używany do:

  • fizyki (Update samochodu),
  • animacji UI (odliczanie),
  • animacji menu (obrót auta).

Maszyna stanów gry: logika per-frame.

Każdy stan ma własną logikę aktualizacji:

  • Splash: odlicza timer i przechodzi do menu,
  • Menu: animuje obrót samochodu / tła,
  • Racing: fizyka, AI, timer wyścigu, naliczanie pieniędzy.

Dynamika dźwięku silnika.

Pitch zależy od aktualnej prędkości:

  • wolno -> niski pitch,
  • szybko -> wysoki pitch.

Logika timera wyścigu.

Jeśli timer aktywny, odejmujemy czas i kończymy wyścig przy 0.

Logika gry właściwej.

Jeśli mamy odliczanie lub animację GO, sterowanie jest blokowane. W przeciwnym razie:

  • przetwarzamy input,
  • aktualizujemy fizykę gracza,
  • aktualizujemy AI,
  • obsługujemy okrążenia, meta, kolizje.

Aktualizacja AI.

AI:

  • wybiera waypoint,
  • steruje skrętem i gazem w zależności od kąta do celu,
  • aktualizuje fizykę,
  • ogranicza prędkość do MaxSpeed.

Ograniczenie prędkości gracza (twardy clamp).

Okrążenia AI.

AI zalicza okrążenie, kiedy: 1) opuści strefę startu (aiLeftStartZone), 2) wróci do strefy.

Okrążenia gracza.

Dodatkowo gracz musi jechać „do przodu” (anti reverse lap farming).

Kolizje toru kartingowego.

W trybie selectedTrack == 2 używamy TrackCollision do cofnięcia auta przy kontakcie ze ścianą.

Debug log pozycji (klawisz P).

logTimer ogranicza spam do ~5 logów/s.

Sekcja „pre-start”.

Odliczanie:

  • raceCountdownActive: zmniejszamy raceCountdown,

Animacja „GO!”.

Aktualizacja kamery.

  • W wyścigu: chase/cockpit,

Render sceny 3D do FBO.
